# 编码规范与性能要求 (Coding Standards)

## 1. XML注释
- 所有的公开方法与类的 XML 注释必须有 `<summary>` 和 `<param>` 标签。
- 使用中文编写要求清晰易懂。

## 2. 异常与安全处理
- 所有游戏 API 调用必须用 `try-catch` 包裹，防止游戏崩溃或插件报错。
- **不允许空 catch**，必须记录日志或提供安全回退。

## 3. 性能底线
- **计算时间**: 每帧决策计算耗时必须 **< 5ms**。
- **内存分配**: 热路径必须避免堆分配 (Heap Allocation)，强制使用 `Span<T>` / `stackalloc` 处理临时数据。
- **数据缓存**: 必须建立合理的机制缓存频繁访问的游戏数据，避免高频度重复计算或读取。
